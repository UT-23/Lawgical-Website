# Redesign Brief for the Existing Law Firm Website

## Objective
Refine the existing law firm website into a premium, highly interactive, and trustworthy digital experience while preserving all current content, routes, forms, and business logic.

## What must stay the same
- Existing copy and legal content
- Navigation structure
- Pages and routes
- Form behavior
- SEO metadata
- Core business logic
- Brand voice

## What should change
- Visual hierarchy
- Section styling
- Motion design
- Hover states
- Microinteractions
- Premium layout rhythm
- Contact page interactions
- Practice area presentation
- Attorney profile presentation

## Design direction
The site should feel like:
- a premium law firm
- a polished product website
- a highly credible advisory brand

## Inspiration language
Borrow interaction patterns from:
- KodyTechnolab for image-reveal service cards and location panels
- React Bits for animated text and scroll reveal
- Uiverse for buttons, hover cards, accordions, and segmented controls

## High-priority upgrades
### Homepage
- Animated hero headline
- Better CTA hierarchy
- Practice-area cards with hover image reveal
- Animated case metrics
- Premium testimonials section

### Practice areas
- Numbered cards
- Image reveal on hover
- Clear practice descriptions
- Elegant CTA treatment

### Attorneys
- Hover-reveal profile cards
- Modal or expanded bio experience
- Strong typography and spacing

### Contact page
- Service chips instead of basic dropdowns
- Better form focus states
- Expanding office location panels
- More premium lead capture layout

### Navigation
- Smooth dropdown motion
- Practice preview cards
- Cleaner mega menu hierarchy

## Interaction principles
- Motion should be subtle and intentional
- Hover states should reward exploration
- State changes must feel smooth and premium
- Every interaction should improve clarity, trust, or engagement

## Output expectation for Antigravity
Generate:
- component-by-component redesign plan
- updated design system
- motion system
- implementation architecture
- code-ready UI proposals
- page-specific redesign notes

## Non-goals
- Do not rebuild the site from scratch
- Do not change legal content or copy
- Do not add distracting visual gimmicks
